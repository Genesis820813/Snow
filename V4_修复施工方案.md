# Snow V4 修复施工方案

依据：Aion 三路深审（2026-07-17，44 文件全读）+ Grok 全量只读审核报告（桌面 docx）合并去重。
目标：修完后 V4 达到「可打包、可分发、可开源」状态。

## 施工铁律（每批次都遵守）

1. 一步一步改，一次只动一项，X先生喊停就停
2. 优先 patch，不整文件重写
3. 每项改完：`py_compile`（doraise=True）验证 → `find /d/snow_v4 -name '*.py' -path '*/snow/*' -exec touch {} +` 刷 pyc → 杀 pythonw.exe → X先生启动验证
4. subprocess 一律 creationflags=0x08000000
5. 所有 open() 显式 encoding="utf-8"

## 批次1：运行期硬伤（现在修，不依赖打包）12项

### F01 托盘菜单 GC 崩溃 [P0]
- 位置：window.py:1022-1024
- 问题：`m = QMenu()` 无 parent 无引用，setContextMenu 不接管所有权，GC 后右键托盘 = segfault
- 修法：`self._tray_menu = QMenu()` 持久引用；顺手把 :1021 tooltip "Snow v3" 改 "Snow"
- 验证：启动后右键托盘图标，菜单正常弹出、退出可用

### F02 日志重定向 GBK 编码炸弹 [P0]
- 位置：main.py:20-21（stdout/stderr）、:34（console.log）、:87（diag.log）
- 问题：open() 未指定 encoding，Windows 默认 GBK，任何中文/emoji traceback 触发 UnicodeEncodeError 二次崩溃
- 修法：四处补 `encoding="utf-8", errors="replace"`（stdout/stderr 加 buffering=1 行缓冲）
- 验证：启动后 python -c 触发一条含中文的 print，检查 stdout.log 正常写入

### F03 worklog 坏文件 → UI 永久"思考中" [P1]
- 位置：worklog.py:36（mark_last_done）、:29（get_recent）
- 问题：json.loads 无容错，work_log.json 一旦损坏，ai_done 信号在 emit 前抛异常，每轮对话必现
- 修法：两处包 try/except，损坏时重置为 []，并写 diag 日志
- 验证：手动把 work_log.json 写坏 → 对话一轮 → UI 正常结束且文件被重置

### F04 主线程同步网络请求周期性冻结 UI [P0]
- 位置：window.py:610-612（15s 定时器）→ weather.py:8 / hotsearch.py:12
- 问题：断网/缓存过期时最坏卡 UI 30-40 秒，每 15 秒一次；且失败不写缓存时间戳，持续重试
- 修法：_refresh_cards 改 QThreadPool.globalInstance().start(runnable)，取数在后台线程，结果经信号回主线程刷卡片；weather/hotsearch 失败也刷新缓存 ts（失败后至少隔 5 分钟再试）
- 验证：断网启动，UI 拖动流畅无卡顿；恢复网络后下轮刷新数据回来

### F05 world.py sys 遮蔽 → chat_only 检查死代码 [P1]
- 位置：tools/world.py:96-107
- 问题：:107 函数内 `import sys` 使 :97 的 sys.path.insert 抛 UnboundLocalError 被吞，极简主题检查永远不执行，工具谎报渲染成功
- 修法：删掉函数内 import sys，模块顶部统一 import；顺手修文件首行 docstring "閳" 乱码（Grok 发现）
- 验证：chat_only 主题下让 AI 调世界工具，应返回"没有世界面板"而非假成功

### F06 agent 线程无顶层兜底 → 静默死 [P0]
- 位置：agent.py:111-133（_agent_loop 循环体前半在 try 之外）
- 问题：build_system_prompt/get_tool_defs/json.dumps 任一抛异常，QThread 静默死亡，UI 永久转圈
- 修法：_agent_loop 整体包 try/except，异常时 emit error_signal + 写 diag
- 验证：临时把 prompts 目录改名触发异常 → UI 显示错误提示而非卡死

### F07 历史裁剪拆散 tool_calls 配对 → 永久 400 [P1]
- 位置：agent.py:107-108
- 问题：按条数硬切可能把 assistant(tool_calls) 和 tool 回包拆开，孤儿 tool 消息导致 DeepSeek 每次 400，不重启不自愈
- 修法：裁剪后循环检查——若首条是 role=tool 或首条 assistant 带未配对 tool_calls，继续向后丢弃至干净边界
- 验证：构造 350+ 条含工具调用的历史，裁剪后首条必为普通 user/assistant

### F08 todos 并发丢数据 + 非原子写 [P1]
- 位置：todos.py:21-47（agent 线程与主线程并发读改写）
- 问题：丢更新；写一半崩溃 → _load 返回 [] 静默清空全部待办
- 修法：模块级 threading.Lock 包全部读改写；写入改 tmp 文件 + os.replace 原子替换。memory.py:19、worklog.py 同样改原子写并去掉静默吞异常
- 验证：脚本并发 add/toggle 100 次，无丢失无损坏

### F09 TTS 三处竞态 [P1]
- 位置：tts.py:92-94（_generating 检查置位不在锁内）、:142-155（双 _play_all 并发 → 音频重叠）、:174-189（Popen 抛异常时 proc 未绑定 → finally NameError 播放线程崩死）、:195-209（_stop_playback 不清 _wavs）
- 修法：① _generating 检查+置位放进锁内；② _play_all 入口加 _playing 原子检查，保证单实例；③ proc = None 先行绑定；④ _stop_playback 清空 _wavs/_wav_idx 并删除残留 mp3
- 验证：连续快速发 5 条消息让 TTS 排队，无重叠、无卡死；中途打断无残留文件

### F10 世界面板系统项点击全灭 [P1]
- 位置：panels/world.py:156、:229
- 问题：此电脑/回收站等 target 是带参数命令行（"explorer.exe /e,::{...}"），os.startfile 必抛 FileNotFoundError 且槽内未捕获
- 修法：target 含空格/参数时走 `subprocess.Popen(target, shell=False, 拆分参数, creationflags=0x08000000)`；整个点击槽包 try/except；顺手补 :100-108 show_icons 先 clear 旧图标、:137 dict 取值改 .get()
- 验证：世界面板点"此电脑""回收站"正常打开

### F11 read_file 无大小限制 [P1]（Grok 发现）
- 位置：tools/file.py:16-26
- 问题：先整读进内存再截 4000 字，大文件内存爆炸；且仅试 utf-8，GBK 文本一律读不了
- 修法：先 os.path.getsize，>10MB 拒绝并告知；打开失败回退 encoding="gbk"
- 验证：让 AI 读一个 500MB 文件 → 得到"文件过大"；读一个 GBK txt → 正常

### F12 run_shell 中文乱码误导模型 [P1]
- 位置：tools/system.py:17-18、:34-42
- 问题：强制 utf-8 解码 GBK 控制台输出，中文全乱码；psutil fallback 把磁盘剩余当内存展示
- 修法：命令前缀注入 `[Console]::OutputEncoding=[Text.Encoding]::UTF8;`；fallback 解析对齐字段或直接删掉 fallback（psutil 在 requirements 里）
- 验证：run_shell 执行 `dir C:\` 输出中文正常

## 批次2：打包适配 + 安装器（7项，一起做）

### P01 统一路径体系（最大工程，其他项的地基）
- 问题：PROJECT_DIR = Path(__file__) 散落 agent/bridge/tts/weather/todos/memory/worklog/panels/prompt/theme_manager/ui_loader/monitor 等 10+ 处，config/data/日志全写安装目录 → Program Files 下启动即崩
- 修法：新建 snow/paths.py，全项目唯一路径源：
  - `app_dir()`：只读资源（prompts/themes/assets）。dev = 项目根；frozen = sys.executable 所在目录
  - `data_dir()`：可写数据（config/data/logs）。dev = 项目根；frozen = %APPDATA%/Snow
  - `ensure_data_dir()`：首启把默认 config/prompts/themes 模板复制到 %APPDATA%/Snow
  - 判据：`getattr(sys, "frozen", False)`
  - 全项目替换 PROJECT_DIR 引用，日志文件（stdout/stderr/diag/crash/console.log）全部改到 data_dir()/logs/
- 验证：dev 模式行为不变；模拟 frozen（临时设 sys.frozen）文件全部落 %APPDATA%

### P02 重启机制 frozen 适配 + 轮询 PID（合并 Grok 的 sleep(3) 问题）
- 位置：window.py:770-776、_restart_helper.py
- 修法：重启入口统一成一个函数：dev = `Popen([pythonw, main.py, "--wait-pid", str(os.getpid())])`；frozen = `Popen([sys.executable, "--wait-pid", str(os.getpid())])`。main.py 顶部解析 --wait-pid，用 ctypes OpenProcess 轮询旧进程退出（上限 15s），替代固定 sleep(3)。_restart_helper.py 退役
- 验证：dev 下 chat_only↔morandi 切换重启正常；frozen 打包后同样验证

### P03 TTS worker frozen 适配
- 位置：services/_tts_engine.py:24
- 问题：sys.executable 跑 .py，打包后递归拉起主程序，TTS 全灭
- 修法：main.py 最顶部（Qt 导入前）检测 `--tts-worker` 参数，命中则直接 exec _edge_tts_worker 逻辑并退出；_tts_engine dev 下保持现状，frozen 下改调 `[sys.executable, "--tts-worker", ...]`
- 验证：打包后语音播报正常，任务管理器无递归 Snow.exe

### P04 prompts 缺失静默空 prompt
- 位置：prompt.py:8-10
- 修法：目录缺失时写 diag 告警 + 返回内置最小人格 prompt 兜底
- 验证：改名 prompts 目录启动，AI 仍有基本人格且 diag 有告警

### P05 monitor 盘符错位
- 位置：services/monitor.py:32
- 修法：`Path(__file__).drive` 改为 data_dir() 所在盘 + 系统盘
- 验证：打包后监控显示正确磁盘

### P06 PyInstaller spec
- onedir + --noconsole；datas：assets/prompts/themes/config 模板；hiddenimports：edge_tts、win32com、pythoncom；图标 assets/make_icon.py 产物
- 验证：dist/Snow/Snow.exe 双击完整跑通对话+TTS+主题切换

### P07 Inno Setup 安装器
- 装 Program Files；开机自启可选（HKCU Run）；首启无 key 时自动弹设置窗口的 key 页（引导录入）；卸载时询问是否清理 %APPDATA%/Snow
- 验证：干净虚拟机/另一台机安装 → 首启引导 → 全功能可用 → 卸载干净
